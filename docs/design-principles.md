# 綠洲所 Design Principles

> 產品設計原則 · 品牌調性 · 視覺語言定義

---

## 品牌核心

**綠洲所 Oasis Studio** 是一個面向 20–30 歲都市租屋族的植物電商平台。品牌相信：即使在最小的空間裡，一盆植物也能改變一個人與生活的關係。

**品牌個性關鍵字：** 沉穩、清新、有溫度的專業感
**不是：** 可愛風、極簡冷調、戶外園藝粗獷感

---

## 設計原則

### 1. 安靜的信任感 Quiet Confidence

> 讓產品自己說話，設計退到背後。

綠洲所的使用者多數是植物新手，他們需要的不是華麗的視覺刺激，而是一種「這裡的人懂植物，我可以放心買」的感覺。設計應該傳遞專業但不冷漠、自信但不張揚的氣質。

**實踐方式：**
- 以大面積留白和克制的色彩使用建立呼吸感，而非填滿每一寸空間
- 資訊架構清晰，讓使用者能快速找到他們需要的判斷依據（光照、澆水、寵物友善）
- 避免過度裝飾——沒有漸層、沒有多餘的圖示、沒有不承載功能的裝飾元素
- 字體選擇傳遞質感：Noto Serif TC 用於標題帶出沉穩感，JetBrains Mono 用於數據帶出精確感

### 2. 有溫度的直角 Warm Precision

> 銳利的結構，柔軟的內容。

全站使用直角（border-radius: 0）作為統一的視覺語言。這不是冷硬的工業感，而是像一本精裝書、一個手工陶盆——有稜角但觸感溫潤。銳利的結構框架裡，裝的是溫暖的植物照片、有人味的文案、和圓潤的手繪感 icon。

**實踐方式：**
- 所有容器、卡片、按鈕、輸入框統一使用 0 圓角
- 溫度來自內容而非形狀：真實植物攝影、口語化文案、emoji 配送說明
- 色彩以自然界的綠色和大地色為主，避免高飽和的數位感色彩
- 森綠 (#1f3a2e) 作為主色調，沉穩但不沉重；配合米白底色 (#fafaf6) 營造紙張觸感

### 3. 漸進式揭露 Progressive Disclosure

> 不要一次說完所有事，讓好奇心帶路。

植物新手容易被過多資訊嚇到。設計應該先給出最關鍵的判斷依據（價格、外觀、難易度），把細節藏在可探索的層級裡。使用者不需要知道所有事，只需要在想知道的時候能找到。

**實踐方式：**
- 首頁：Hero → 促銷 → 精選商品 → 分類 → 情境 → 評價 → 指南，從感性到理性的敘事節奏
- 商品卡片預設只顯示名稱和價格，hover 才展開光照/澆水等屬性 badge
- 商品頁使用 Tab 分隔養護指南/規格/配送/評價，不一次攤開
- 情境推薦（給辦公室/給小套房/給新手）比商品分類更先出現——先解決「我適合什麼」再解決「有什麼」

### 4. 一致的節奏感 Consistent Rhythm

> 每個區塊都在呼吸，但頻率一致。

頁面應該有韻律——標題的大小、區塊的間距、grid 的疏密感，都遵循可預期的節奏。使用者不需要意識到這種節奏，但當節奏被打破時他們會感到不舒服。

**實踐方式：**
- 每個區塊遵循統一結構：Eyebrow (Mono 11px) → 標題 (Serif 32px) → 內容 grid → 留白
- 三級字體系統各司其職：Serif 負責情感（標題、引言）、Sans 負責資訊（內文、說明）、Mono 負責數據（價格、標籤、倒數）
- 側邊 padding 遵循斷點階梯：56px (Desktop) → 32px (Tablet) → 20px (Mobile)
- 色彩透明度層級有明確的語意：ink-60 = 輔助文字、ink-40 = 次要資訊、ink-15 = 分隔線、ink-08 = 邊框

### 5. 觸手可及的行動 Effortless Action

> 每個想買的瞬間，按鈕都在手邊。

電商的核心是轉換。設計不應該讓使用者在想要行動的時候找不到出路。但行動呼籲不應該是侵略性的——它應該是自然的、可預期的、就在那裡。

**實踐方式：**
- 按鈕三級系統語意明確：Primary（深綠實心）= 主要行動 → Secondary（描邊）= 次要行動 → Ghost（文字）= 探索性連結
- 商品卡片右下角永遠有一個加入購物車的圓形按鈕，不需要進入商品頁就能加購
- 願望清單愛心永遠在右上角，位置可預期
- Toast 通知在操作後即時回饋（2.4 秒），確認使用者的行動被接收
- Mobile 按鈕撐滿寬度、加大觸擊面積，不要讓拇指找按鈕

### 6. 為真實空間而設計 Design for Real Spaces

> 這不是一個關於植物的網站，是一個關於「你和植物一起生活」的網站。

使用者買的不是一盆植物，是他們想像中的生活畫面——窗邊有光、桌上有綠、回家有期待。設計應該幫助使用者把植物放進自己的生活情境裡。

**實踐方式：**
- 情境推薦（辦公室/小套房/新手）以使用場景分類，而非植物學分類
- 商品照片優先選擇有空間感的實拍（放在桌上/窗邊/書架旁），而非去背白底棚拍
- 評價設計強調場景：顯示買家的城市和空間（台北・大安區・工程師），讓人產生「跟我一樣」的共鳴
- 養護指南的語氣像朋友聊天，不像教科書：「盆土表面 2cm 乾燥再澆透」而非「維持適當土壤濕度」

### 7. 響應式是原生的 Responsive is Native

> 不是「也能在手機上看」，而是「在每個裝置上都是最好的版本」。

超過半數的使用者會在手機上瀏覽。響應式不是桌面版的縮小版，每個斷點都應該是為那個尺寸量身設計的體驗。

**實踐方式：**
- 使用 container-based breakpoints (ResizeObserver)，而非 window-based media queries——確保元件在任何容器寬度下都正確響應
- Mobile 的導航從水平選單轉為漢堡 + 側邊抽屜，帶有 slide-in 動畫和背景遮罩鎖定
- Hero 按鈕在 Mobile 從橫排改為直排堆疊，確保可觸擊面積
- Tab 在 Mobile 支援橫向滑動，不截斷文字
- 商品卡片 hover 效果（第二張圖、屬性 badge）在 Mobile 不顯示——觸控裝置沒有 hover

## 視覺語言摘要

| 維度 | 定義 |
|------|------|
| **色調** | 森林綠 (#1f3a2e) 為主，大地色系輔助，米白紙感底色 |
| **形狀** | 全站直角 (border-radius: 0)，銳利但不冷硬 |
| **字體** | 三級系統：Serif (情感) / Sans (資訊) / Mono (數據) |
| **留白** | 大方的留白，區塊間有呼吸感 |
| **攝影** | 真實場景 > 去背棚拍，有光線有空間感 |
| **動態** | 克制的微動效：hover 上移 2px、carousel 淡入、toast slide-in |
| **文案** | 口語化、有人味、像朋友推薦而非店員推銷 |
| **圖示** | 線條型 SVG icon，1.4–1.6px stroke，不填充，與文字同色 |
