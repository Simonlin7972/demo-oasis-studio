**Status:** Draft
**Owner:** —
**Last Updated:** 2026-04-28
**Target Release:** —
**Availability:** All users

---

## Problem

當使用者對某盆植物感興趣，他們需要足夠的資訊來做出購買決定——外觀、養護難度、尺寸選擇、價格、其他人的評價。現有的商品卡片只能提供概覽，使用者需要一個完整的商品詳細頁來回答「這盆植物適不適合我？」的問題。

## Target Persona

- 已經在首頁被吸引，正在考慮購買的潛在顧客
- 對特定植物有興趣，需要養護難度、尺寸、價格等細節
- 可能是新手（需要養護指南）或有經驗者（需要規格資訊）

## 現有功能規格

### 1. 路由與導航

| 項目 | 規格 |
|------|------|
| URL 格式 | `?product={productId}`（如 `?product=p1`） |
| 進入方式 | 點擊首頁商品卡片 → pushState 更新 URL |
| 返回 | 麵包屑「首頁」按鈕 / 瀏覽器上一頁（popstate 事件） |
| 直接訪問 | 支援直接用 URL 訪問商品頁，頁面載入時解析 `?product=` 參數 |
| 頁面切換 | `window.scrollTo(0, 0)` 回到頂部 |

### 2. 麵包屑 (Breadcrumb)

| 項目 | 規格 |
|------|------|
| 格式 | 首頁 / 商品 / {商品名稱} |
| 首頁連結 | 可點擊，觸發 `onBack` 返回首頁 |
| 當前頁 | forest-900, fontWeight 500 |
| 分隔符 | "/" ink-40 |
| 字級 | 12px |

### 3. 商品圖片 Gallery

| 項目 | 規格 |
|------|------|
| 主圖 | 1:1 正方形，`object-fit: cover`，sage-100 背景 |
| 縮圖列 | 主圖下方，等分 grid，間距 10px |
| 縮圖數量 | 3 張（主圖 + 第二張 + 主圖重複） |
| 選中狀態 | 2px solid forest-800 border |
| 未選狀態 | 2px solid transparent border |
| Tag | 左上角商品標籤（與卡片一致） |
| 願望清單 | 右上角 38x38 圓形按鈕，toggle 愛心 |

### 4. 商品資訊區

| 項目 | 規格 |
|------|------|
| 版面 | Desktop: 左圖 (1.05fr) 右資訊 (1fr) / Tablet+Mobile: 單欄堆疊 |
| 間距 | Desktop: 56px / Mobile: 24px |

**資訊區域內容（由上至下）：**

| 區塊 | 規格 |
|------|------|
| Tag + 分類 | JetBrains Mono, 11px, forest-600, "熱賣 · 室內植物" 格式 |
| 商品名稱 | Noto Serif TC, Desktop 44px / Mobile 32px, SemiBold |
| 學名 | JetBrains Mono, 12px, italic, ink-40 |
| 評分 | 5 顆星星 (forest-600) + "4.9 · 184 評價" Mono 12px |
| 價格 | JetBrains Mono, 32px, SemiBold, forest-800 |
| 原價 | 刪除線, 14px, ink-40（僅有折扣時顯示） |
| 商品描述 | 14px, line-height 1.85, ink-60, 固定文案 |

### 5. 屬性卡片

| 項目 | 規格 |
|------|------|
| Grid | Desktop: 3 欄 / Mobile: 2 欄 |
| 背景 | cream-50 |
| 內容 | Icon + 標籤 (Mono 10px) + 數值 (13px SemiBold) |
| 屬性 | 光照（需求等級）、澆水（頻率）、寵物（友善/需注意） |

### 6. 尺寸選擇

| 項目 | 規格 |
|------|------|
| 選項 | S 小 (15cm 盆) / M 中 (20cm 盆) / L 大 (25cm 盆) |
| Grid | 3 欄等分，間距 8px |
| 選中狀態 | forest-800 背景 + cream 文字 + 1.5px border |
| 未選狀態 | paper 背景 + ink 文字 + ink-15 border |
| 標題 | 「選擇尺寸」+ 「尺寸對照表 →」連結 |
| 預設 | M |

### 7. 數量與購買

| 項目 | 規格 |
|------|------|
| 數量選擇器 | −/+按鈕 + 數字顯示，pill 型圓角邊框，高度 48px |
| 最小數量 | 1 |
| 加入購物車 | btn-primary，flex: 1 撐滿剩餘寬度，含 cart icon |
| 立即購買 | btn-secondary，width 100%，置中文字 |
| Toast | 加入成功顯示 "已加入購物車：{商品名}" |

### 8. 配送說明

| 項目 | 規格 |
|------|------|
| 背景 | sage-100 |
| 內容 | 3 項目，emoji icon + 說明文字 |
| 項目 | 48hr 保鮮配送、一年植物保固、滿 1200 免運 |
| 分隔 | 1px solid ink-08（第一項無上分隔） |

### 9. Tabs 內容區

| 項目 | 規格 |
|------|------|
| Tab 數量 | 4 個：養護指南、商品規格、配送與退換、評價 (184) |
| Active 狀態 | forest-900 文字 + fontWeight 600 + 2px solid forest-800 底線 |
| Inactive 狀態 | ink-60 文字 + fontWeight 400 + transparent 底線 |
| 預設 Tab | 養護指南 |

**各 Tab 內容：**

| Tab | 內容 | 版面 |
|-----|------|------|
| 養護指南 | 3 張卡片（光照/澆水/濕度），cream-50 背景，含編號 01/02/03 | Desktop 3 欄 / Mobile 1 欄 |
| 商品規格 | Key-value 表格（學名/尺寸/原產地/溫度/毒性/包裝），120px label 欄 | 最大寬度 600px |
| 配送與退換 | 純文字段落（配送範圍/運費/退換政策） | 最大寬度 640px |
| 評價 | 評價卡片 grid，含姓名/地點/星等/引言 | Desktop 2 欄 / Mobile 1 欄 |

### 10. 相關商品推薦

| 項目 | 規格 |
|------|------|
| 標題 | 「你可能也會喜歡」Serif 28px (Mobile 22px) |
| Grid | Desktop: 4 欄 / Tablet: 3 欄 / Mobile: 2 欄 |
| 內容 | 排除當前商品，取前 4 項 |
| 卡片 | 簡化版：圖片 (1:1) + 名稱 + 價格，可點擊 |
| 連結 | `?product={id}` 直接跳轉 |

### 11. 頁面結構

```
ConservativeNav
  └ Breadcrumb
  └ Product Section (Gallery + Info)
  └ Tabs Section
  └ Related Products
  └ ConservativeFooter
```

## 未來功能開發

### Phase 1 — 購買體驗完善

#### 1.1 尺寸與價格連動
- 不同尺寸對應不同價格（S/M/L 各自定價）
- 切換尺寸時價格即時更新
- 庫存狀態顯示（有貨 / 即將售罄 / 缺貨）
- 缺貨時按鈕改為「到貨通知」

#### 1.2 圖片 Gallery 強化
- 每個尺寸對應不同的實拍照片
- 圖片放大鏡（hover 放大局部）
- 全螢幕 lightbox 瀏覽
- 支援影片（養護示範/開箱短片）
- 360 度旋轉檢視

#### 1.3 加購推薦
- 「搭配購買」區塊：推薦陶盆 + 底盤 + 土壤
- 組合價優惠
- 一鍵加入整組
- 「買了這盆的人也買了...」推薦

#### 1.4 配送資訊動態化
- 根據使用者地區顯示預估到貨時間
- 即時物流狀態查詢
- 配送時段選擇（避開高溫時段）
- 冷藏配送選項切換

### Phase 2 — 信任與決策支援

#### 2.1 評價系統完整版
- 評價撰寫功能（僅限已購買使用者）
- 照片/影片上傳
- 養了 N 天後的追蹤評價
- 賣家回覆
- 「此評價是否有幫助？」投票
- 按星等/時間/有照片篩選
- 評價摘要（AI 產生的優缺點統整）

#### 2.2 養護難度視覺化
- 養護難度量表（1-5 星）
- 與其他植物的比較表
- 「適合我嗎？」互動式小測驗
- 每月養護日曆（何時澆水/施肥/換盆）

#### 2.3 AR 預覽
- 手機 AR 模式：在自己的空間中預覽植物大小
- 支援不同尺寸 S/M/L 的 3D 模型
- 搭配不同陶盆的組合預覽

### Phase 3 — 社群與內容

#### 3.1 買家秀 Gallery
- 真實顧客的植物養成照
- 照片標記養了幾天/幾個月
- 空間標記（客廳/書桌/陽台）
- 顧客可以追蹤彼此的養護日記

#### 3.2 養護指南深度化
- Tab 內的養護指南改為互動式內容
- 季節性養護提醒（春夏秋冬不同建議）
- 常見問題 FAQ（葉片發黃？根部腐爛？）
- 影片教學嵌入
- 連結到部落格完整文章

#### 3.3 社群分享
- 商品頁分享按鈕（LINE / Facebook / 複製連結）
- 分享時自動產生 OG image（商品圖 + 名稱 + 價格）
- 「送禮」功能 — 分享給朋友，附禮物訊息

### Phase 4 — 轉換率優化

#### 4.1 限時優惠
- 商品頁內的限時折扣倒數
- 「最後 N 件」庫存緊迫提示
- 首購優惠碼自動帶入
- 瀏覽器通知：「你收藏的商品降價了」

#### 4.2 比較功能
- 「加入比較」按鈕
- 浮動比較列（最多 3 盆）
- 比較表：價格/光照/澆水/尺寸/寵物友善 並列
- 幫助「琴葉榕 vs 龜背芋」這類猶豫型決策

#### 4.3 個人化推薦
- 基於瀏覽/購買歷史的推薦
- 「根據你的空間，我們推薦...」
- 相關商品改為 ML-based 推薦
- 最近瀏覽記錄

#### 4.4 結帳流程
- 單一商品「立即購買」→ 快速結帳頁
- 收件資訊表單（地址/電話/備註）
- 金流整合（信用卡 / LINE Pay / Apple Pay）
- 訂單確認頁 + email 通知
- 訂單追蹤頁

## Open Questions

| 問題 | 假設 | 驗證方式 |
|------|------|----------|
| 使用者會切換尺寸嗎？還是大多選預設 M？ | 尺寸選擇有助決策 | 追蹤尺寸切換率 |
| 養護指南 vs 商品規格，哪個 tab 更常被點？ | 新手偏好養護指南 | Tab 點擊率分析 |
| 相關商品推薦的點擊率？ | 有交叉銷售效果 | 追蹤推薦區 CTR |
| 是否需要「加入購物車」和「立即購買」兩個按鈕？ | 兩個按鈕覆蓋不同意圖 | A/B test 單按鈕 vs 雙按鈕 |
| 評價數量對轉換率的影響？ | 越多評價越有信任感 | 對比有/無評價商品的轉換率 |
